Тестовое задание
Создать REST API модуль "Комментарии к задачам" для CRM-системы. 

Технологии:
- Node.js 
- NestJS
- TypeORM
- PostgreSQL
- TypeScript

Задача 1: Реализовать минимальную реализацию авторизации и регистрации пользователя с JWT токеном.

Сущность Users
id | UUID | Уникальный идентификатор 
password | string | Пароль пользователя
role | string | Роль пользователя (Роль может быть либо автор либо пользователь)
task_id | UUID | ID задачи 
created_at | timestamp | Дата создания 
updated_at | timestamp | Дата обновления 

Эндпоинты:
POST /users - Создать пользователя
GET /users/:id - Список пользователей по ID
GET /users/ - Получить список всех пользователей 
PATCH / users /:id - Редактировать пользователя
DELETE / users /:id - Удалить пользователя

Задача 2: Создание задачи

Сущность Tasks
id | UUID | Уникальный идентификатор 
user_id | UUID | ID пользователя	 
description | string | Текст описания задачи (1-1000 символов) 
created_at | timestamp | Дата создания 
updated_at | timestamp | Дата обновления 
сomment | string | Комментарии данной задачи (1-1000 символов) 

Эндпоинты:
POST /tasks - Создать задачу 
GET / tasks/:id - Список задач по ID
GET / tasks/ - Получить список всех задач 
PATCH / tasks/:id - Редактировать задачу 
DELETE / tasks/:id - Удалить задачу 

Бизнес-правила:
1. Создавать, редактировать, удалять комментарий может только его пользователь
2. Задачи возвращаются отсортированными по дате (новые первыми)
3. Обязательно реализовать между ними связь у каждой задачи может быть несколько комментариев и комментарий может быть привязан только к одной задаче
4. Только пользователь может создать задачу


Задача 3: Реализовать CRUD для комментариев к задачам. 

Сущность Comment
id | UUID | Уникальный идентификатор 
task_id | UUID | ID задачи 
user_id | UUID | ID пользователя	 
text | string | Текст комментария (1-1000 символов) 
created_at | timestamp | Дата создания 
updated_at | timestamp | Дата обновления 

Эндпоинты:
POST /comments - Создать комментарий
GET /comments?task_id=xxx - Список комментариев к задаче 
GET /comments/:id - Получить комментарий по ID
PATCH /comments/:id - Редактировать комментарий
DELETE /comments/:id - Удалить комментарий

Бизнес-правила:
1. Редактировать и удалять комментарий может только его пользователь
2. Текст комментария обязателен (1-1000 символов)
3. Комментарии возвращаются отсортированными по дате (новые первыми)
4. Только автор может создать комментарий

Требования к сдаче:
- Код на GitHub
- README с инструкцией по запуску, аргументировать свои действия иначе тестовое задание будет аннулировано 
- docker-compose.yml для PostgreSQL

Будет плюсом: Swagger документация, создание refresh токена, валидация данных
